# Business Project Screening Templates

## Structured Intake Template

Use this before scoring when the user's idea is too vague, or when missing facts would materially change the conclusion.
Present it as a direct fill-in form. Do not add long explanations under each field unless the user asks.

```text
欢迎使用 AES 的一人公司筛选器。

Intake（请直接填写）:
如果暂时不清楚，就写“模糊”或“不知道”。

1. 你的目标用户是：（用户填写）

2. 他们现在最具体的痛点或损失是：（用户填写）

3. 他们现在通常怎么解决这个问题/已有替代方案是什么：（用户填写）

4. 你希望用户为什么结果付费：（用户填写）

5. 你准备怎么找到前 10 个真实用户：（用户填写）

6. 不做产品/不开店时，你能先人工交付什么：（用户填写）

Optional:
7. 你打算测试的价格是：（用户填写）
8. 你为什么适合做这件事：（用户填写）
9. 你现在最担心的问题是：（用户填写）
```

## Full Review Template

```text
Conclusion（结论）: continue / pause / stop

Input boundary（输入边界）:
{Only use facts stated in the current project brief. Mark missing facts as unknown（未知）. Do not carry over facts from earlier examples or tests.}

One-sentence judgment（一句话判断）:
{What this project is really trying to solve, and whether it fits one-person validation.}

Quadrant（象限区域）:
{A / B / C / D. For one-person/vibe-coding/tool-first projects, explain why. If current product shape and recommended validation shape differ, show both.}

Traffic-light logic（红黄绿灯运行逻辑）:
{Red lights decide whether to stop or pause; yellow lights show what must be shrunk; green lights only justify a 3-7 day paid validation, not a full build.}

Count-to-decision（数量判定）:
{Map the light counts to the conclusion. Example: Red >= 3 -> stop; Red 1-2 -> pause; Red 0 + Yellow >= 3 -> pause; Red 0 + Yellow <= 2 + Green >= 3 -> continue.}

Normalized project（项目标准化）:
- Target user（目标用户）:
- Scene（使用场景）:
- Current alternative / existing substitute（当前替代方案/已有替代方案）:
- Possible payer（可能付费人）:
- Paid deliverable（可收费交付物）:
- Acquisition path（获客路径）:
- Project shape（项目形态）:

Scores（评分）:
- Monetization Priority Index（变现优先级评分）: {0-100}
- 7-30 Day Paid Validation Probability（7-30 天付费验证概率）: {0-100%}

Red lights（红灯）: {count}
- {risk}

Yellow lights（黄灯）: {count}
- {uncertainty to shrink}

Green lights（绿灯）: {count}
- {evidence or advantage}

Six-question audit（六问审查）:
- Demand（需求）:
- Payment（付费）:
- Alternative（替代方案）:
- Reach（触达）:
- Delivery（交付）:
- Accumulation（积累）:

Biggest false assumption（最大错误假设）:
{The one assumption most likely to kill the project.}

Biggest opportunity variable（最大机会变量）:
{The fact that, if proven true, most improves the project.}

Fastest validation（最快验证）:
1. {action}
2. {action}
3. {action}

Success threshold（成功标准）:
{What must happen to continue.}

Stop condition（停止条件）:
{What result means stop or radically shrink.}

Only do next（下一步只做这件事）:
{The single next action.}

Disclaimer（免责声明）:
本工具仅做辅助判断，不替你做最终决策，也不对你的商业、法律、财务、健康或实际经营结果负责。
```

## Offer Test Template

```text
Target（目标用户）:
{specific user}

Offer（付费交付物）:
{specific paid result}

Price（测试价格）:
{real price}

Message（触达话术）:
"I am testing a {service/tool}. It helps {target user} solve {problem} by delivering {result}. The test price is {price}. I only need {input} from you and will deliver {output} by {time}. Would you pay for this?"

Success（成功标准）:
{payment/deposit/accepted price count}

Stop（停止条件）:
{rejection or non-response threshold}

Disclaimer（免责声明）:
本工具仅做辅助判断，不替你做最终决策，也不对你的商业、法律、财务、健康或实际经营结果负责。
```
